# goal-misgeneralization-rl

RL-first variant for studying goal misgeneralization in Nim-like games.

This version replaces supervised answer-token fine-tuning with game-play RL.
The model plays full episodes, receives win/loss reward, and is evaluated under
opponent-distribution shift. This makes the proxy story a reward story: a policy
such as “play a mod-2 coset move” can be highly reward-correlated against a
weak/random opponent, then fail against an optimal opponent.

The implementation supports:

- bounded Nim: modular/coset proxies such as `coset2`, `coset3`, `coset4`
- multipile Nim: XOR proxies such as `xor2`, `xor4`
- Fibonacci Nim: aperiodic `fibonacci_floor` proxy
- Wythoff Nim: aperiodic `balance` and `difference` proxies

## Train Against A Weak Opponent

Run commands from this directory with the parent project on `PYTHONPATH`:

```bash
cd rl_version

PYTHONPATH=../src python scripts/run_train.py \
  --game bounded \
  --mr 5 \
  --model 410m \
  --train-opponent random \
  --eval-opponents random optimal coset2 \
  --output-dir runs/bounded_mr5_random_train \
  --steps 30000 \
  --episodes-per-step 8 \
  --learning-rate 1e-6 \
  --kl-coef 0.02 \
  --proxy-metrics coset2 coset3 \
  --bf16
```

Interpretation:

- `win_rate` against `random` is the training-distribution reward proxy.
- `win_rate` against `optimal` is the shifted test condition.
- `coset2_rate`/`coset3_rate` show whether the policy is acquiring a proxy
  ladder rather than the exact optimal rule.
- `optimal_action_rate` tracks actual strategic correctness on winning states.

## Other Games From Appendix F

Multipile Nim, with compositional XOR proxy metrics:

```bash
PYTHONPATH=../src python scripts/run_train.py \
  --game multipile \
  --pile-count 3 \
  --train-opponent random \
  --eval-opponents random optimal xor2 xor4 \
  --proxy-metrics xor2 xor4 \
  --output-dir runs/multipile_random_train
```

Fibonacci Nim, with an aperiodic floor-to-Fibonacci proxy:

```bash
PYTHONPATH=../src python scripts/run_train.py \
  --game fibonacci \
  --train-opponent random \
  --eval-opponents random optimal fibonacci_floor \
  --proxy-metrics fibonacci_floor \
  --output-dir runs/fibonacci_random_train
```

Wythoff Nim, with balance and difference proxies:

```bash
PYTHONPATH=../src python scripts/run_train.py \
  --game wythoff \
  --train-opponent random \
  --eval-opponents random optimal balance difference \
  --proxy-metrics balance difference \
  --output-dir runs/wythoff_random_train
```

## Evaluate A Checkpoint

```bash
PYTHONPATH=../src python scripts/run_eval.py \
  --checkpoint runs/bounded_mr5_random_train/checkpoint-30000 \
  --game bounded \
  --mr 5 \
  --eval-opponents random optimal coset2 \
  --proxy-metrics coset2 coset3 \
  --eval-episodes 1000 \
  --output results/bounded_mr5_shift_eval.jsonl
```

The original paper reports supervised fine-tuning. This directory is the GMG
extension: same games, but reward is generated by play and the key train/test
shift is the opponent distribution.
